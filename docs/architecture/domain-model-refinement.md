# EduInteractive - Domain Model Refinement (Phase 2 Patches)

This document addresses the final architectural gaps identified in the Phase 2 Domain Model audit, introducing the Execution Authority, Event Taxonomy, and Event Dispatcher.

---

## 1. Patch 2.1: LearningExecutionAggregate (Execution Authority)

The `LearningExecutionAggregate` is the root entity responsible for the active runtime execution of learning content. It bridges the gap between static definitions (Lesson/Activity) and the active session state.

*   **Responsibility:** 
    *   Orchestrates the active flow within a `LearningSession`.
    *   Binds the student's active interaction context to specific `LearningObjects`.
    *   Enforces runtime transition rules (e.g., "Cannot proceed until Activity completed").
*   **Attributes:**
    *   `sessionId`: Link to session.
    *   `currentObjectId`: Pointer to the active object (Lesson/Activity).
    *   `executionState`: `[READY, EXECUTING, SUSPENDED, TERMINATED]`.
    *   `sessionContext`: Snapshot of user context at execution time.
*   **Lifecycle:** Orchestrates the transition from `READY` -> `EXECUTING` -> `TERMINATED`.

---

## 2. Patch 2.2: Event Taxonomy Standard

To ensure event determinism, all system events are classified into three strict classes.

| Event Class | Definition | Authority |
| :--- | :--- | :--- |
| **System (Authoritative)** | Represents fundamental state changes (e.g., `SessionStarted`, `ActivityCompleted`). | System Core |
| **Derived (Computed)** | Represents state updates calculated from other events (e.g., `CompetencyUpdated`, `EvidenceBundleCreated`). | Domain Engines |
| **Advisory (Advisory)** | Non-authoritative signals generated by AI/Heuristics (e.g., `ScaffoldingSuggested`, `InterestPredicted`). | AI Collaboration Engine |

*   **Enforcement:** Event consumers must ignore `Advisory` events if they conflict with `System` or `Derived` event state.

---

## 3. Patch 2.3: Domain Event Dispatcher

The `DomainEventDispatcher` acts as the explicit orchestrator for event routing, preventing loosely coupled chaos.

*   **Role:** The single entry point for all events emitted by aggregates/services.
*   **Routing Logic:**
    1.  **Validation:** Verifies event taxonomy and schema.
    2.  **Registration:** Logs the event to the Event Store.
    3.  **Broadcast:** Dispatches to registered subscribers (`CompetencyEngine`, `AnalyticsEngine`, `AICollaborationEngine`, etc.).
    4.  **Error Handling:** Monitors for consumption failure in downstream services.
*   **Importance:** Centralizes the event-driven runtime, making the flow explicit rather than implicit.

---

## 4. Impact on Architecture

With these patches, the `LearningExecutionAggregate` becomes the **"Single Source of Authority"** for the learning journey, while the `DomainEventDispatcher` ensures the **"Coherence of the Event-Driven Runtime"**. The system is now deterministically scalable.
