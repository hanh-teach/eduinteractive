# EduInteractive - Recommendation Engine

The Recommendation Engine is the brain of the Adaptive Learning system. It leverages the state generated by the Competency Engine (Layer 6) and historical Evidence (Layer 5) to dynamically determine the most effective next learning step.

---

## 1. Adaptive Learning Logic

The engine does not follow static paths. It utilizes a **Pedagogical Recommendation Algorithm**:

1.  **State Context:** Fetches `CompetencyState` + `StudentSessionState`.
2.  **Constraint Check:** Applies `AdaptationRule` (e.g., "Must complete Lesson A before B").
3.  **Goal Alignment:** Filters all `LearningActivities` that align with current learning objectives.
4.  **Scoring & Ranking:** Ranks activities based on `ConfidenceScore` (historical performance) and `EngagementSignal` (from Behavior Engine).
5.  **Selection:** Selects the optimal activity.

---

## 2. Event Subscription Model

*   **Subscribes to:** `CompetencyUpdated`, `LessonCompleted`, `AnswerSubmitted` (for immediate scaffolding).
*   **Emits:** `RecommendationGenerated` (Layer 0).

---

## 3. Recommendation Types

| Type | Purpose | Trigger |
| :--- | :--- | :--- |
| **Pathing** | Advance to the next logical `LearningActivity`. | `LessonCompleted` |
| **Scaffolding** | Provide a simpler or alternative path. | `DifficultyThresholdExceeded` |
| **Extension** | Suggest deeper exploration or creative task. | `MasteryLevel` = Mastery |
| **Recovery** | Re-teach concept via different modality. | `ConfusionDetected` |

---

## 4. Software Translation Layer

| Domain Entity | Infrastructure Component | Responsibility |
| :--- | :--- | :--- |
| `AdaptationRule` | `AdaptationPolicyService` | Manages pedagogical branching rules. |
| `LearningPath` | `PathOrchestratorService` | Maintains the runtime view of the student's journey. |
| `Recommendation` | `RecommendationEngineService` | Executes ranking logic and emits `RecommendationGenerated`. |
